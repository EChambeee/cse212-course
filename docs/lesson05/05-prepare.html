The Set Data Structure

Previously we learned about lists, stacks, and queues. The location of each item in these data structures was very important to the proper use of the data structure. Not all data structures worry about the order of the data. The Set data structure is an example of one for which order is not important.

Besides the lack of order, the Set has another difference. Could we add duplicate data to a list, stack, or queue? The answer is yes and this situation is very common. However, the Set is constrained to allow no duplicates. Knowing that there will be no duplicates (and because we don't care about the order) allows us to store the information in a Set so as to make it very efficient to determine if data is in the Set. This test of membership in the Set is the most important operation belonging to this data structure.

Sets in Python

Most Sets implemented in software will have the following basic functions:

    add(value) - Adds "value" to the Set

    remove(value) - Remove the "value" from the Set

    in(value) - Determine if "value" is in the Set

    len() - Return the number of items in the Set

In Python, a Set can be represented using a curly braces or the set class name. The in operator can be used to determine membership in the Set.

The performance of the Set is based on the performance of the hashing algorithm.

Function Description Performance
add(value) Performance of hashing the value (assuming good conflict resolution) O(1)
remove(value) Performance of hashing the value (assuming good conflict resolution) O(1)
int(value) Performance of hashing the value (assuming good conflict resolution) O(1)
len() Performance of returning the size of the set O(1)
