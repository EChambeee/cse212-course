CSE 212 | Programming with Data Structures

04 Teach: Group Activity

Overview

You will pair up with another member in your assigned study group to debug two implementations of a Queue. A good method of finding defects is to review the code as we discussed previously. However, in this activity, you should fight the urge to read the code first. Instead, you will write tests to help identify where the code errors are hiding.

Instructions

Problem 1 - Traditional Queue

  1. Download the following code and load into your code editor: 04-teach-traditional_queue-buggy.py

  2. The Queue class implements the basic functions of enqueue and dequeue. If the queue is empty, then the dequeue function should raise an exception. The code also includes 3 test cases. Run the code and determine which tests fail.

  3. If the test case fails, then you know that there is an error in the function you were running. If Python gives you an error message, you can see which lines of code where trying to execute. Look in the code, find the error, and then fix the code. All your test cases should pass when the code is fixed. There are 2 major errors in the code your were given.

  4. When you are done, take a look at the solution and discuss it together: 04-teach-traditional_queue-fixed.py

Problem 2 - Customer Service Queue

  1. Download the following code and load into your code editor: 04-teach-customer_service-buggy.py

  2. The CustomerService class has the ability to add a new customer (the enqueue) and serve a customer (dequeue). When you create the CustomerService object, you have to specify the maximum number of customers that can be in the queue. The CustomerService class should be robust (i.e. handle gracefully) in the cases of empty or full queues. With this knowledge, write some test cases together and implement them. Remember that a test case should cover one of the possible scenarios that the CustomerService class should be able to execute successfully.

  3. Run your test cases.

  4. If your test case fails, then you know that there is an error in the function you were running. If Python gives you an error message, you can see which lines of code where trying to execute. Look in the code, find the error, and then fix the code. All your test cases should pass when the code is fixed. There are 3 major errors in the code your were given.

  5. When you are done, take a look at the solution and discuss it together: 04-teach-customer_service-fixed.py
