CSE 212 | Programming with Data Structures

04 Teach: Pair Activity

Overview

You will pair up with another member in your assigned study group to debug two implementations of a Queue. In this activity, you should fight the urge to read through all the code first. Instead, you will first write tests to help identify where the code errors are hiding.

Instructions

Problem 1 - Traditional Queue

  1. Download the following code and load into your code editor: traditional_queue.py

  2. The Queue class requirements are as follows:

    • The enqueue function shall put a new item in the Queue
    • The dequeue function shall remove an item from the Queue
    • If the Queue is empty, then the dequeue function shall raise an IndexError exception

  3. The code contains 3 tests cases already implemented for you. Run the code and determine if a test has failed. If the test case fails, then you know that there is an error in the function related to the test. If Python gives you an error message, you can see which lines of code the error occurred on. Find the errors and fix the code. All your test cases should pass when the code is fixed. There are 2 major errors in the code your were given.

  4. When you are done, take a look at the solution and discuss it together: traditional_queue-solution.py

Problem 2 - Customer Service Queue

  1. Download the following code and load into your code editor: customer_service.py

  2. The CustomerService class requirements are as follows:

    • Each record in the Customer Service Queue shall contain the name, account ID, and the problem.
    • The user shall specify the maximum size of the Customer Service Queue when it is created.
    • The add_new_customer function shall add a new customer to the queue.
    • If the Customer Service Queue is full, then the add_new_customer function will display an error message.
    • The serve_customer function shall remove the next customer from the queue and display the details.
    • If the Customer Serivce Queue is empty, then the service_customer function shall raise an exception.

  3. Write test cases based on the requirements above and then implement the tests.

  4. Run your test cases. If the test case fails, then look in the related code for the error and fix it. All your test cases should pass when the code is fixed. There are 3 major errors in the code your were given.

  5. When you are done, take a look at the solution and discuss it together: customer_service-solution.py
